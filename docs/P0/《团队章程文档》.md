# Project Roles and AI Tool Selection

## 1. AI编程助手（开发负责人）

### 候选工具
- GitHub Copilot
- Cursor

### 选择理由
- **GitHub Copilot**：适配主流IDE，生态完善且成本可控，底层模型可便捷切换，团队熟悉度高，能辅助代码补全、审查，贴合协作需求。
- **Cursor**：基于VS Code改造，操作易用，功能精准，擅长核心模块开发与代码重构。

---

## 2. AI对话工具（需求负责人）

### 候选工具
- ChatGPT
- Claude

### 选择理由
- **ChatGPT**：自然语言理解较强，擅长生成结构清晰、可读性强的文本内容。
- **Claude**：编码能力强，便于将对话内容汇入到实践代码中。

---

## 3. AI代码审查工具（架构负责人）

### 候选工具
- Gemini
- Claude

### 选择理由
- **Gemini**：可一次性读取 CampusHub 全量代码，擅长全局架构审查与防耦合，负责宏观的体系结构审查。
- **Claude**：逻辑推理顶尖，擅长精准排查核心业务的深层漏洞，有效保障代码健壮性与审查效率，负责微观的逻辑健壮性诊断。

**协同策略**：双模型交叉验证，确保核心模块的开发质量。

---

## 4. AI测试工具（测试负责人）

### 候选工具
- TestGPT
- Applitools

### 选择理由
- **TestGPT**：可基于需求文档自动生成测试用例与测试数据，支持自然语言生成测试脚本，大幅提升用例设计效率，降低人工遗漏。
- **Applitools**：AI视觉测试能力强，自动识别 UI 布局、样式、交互差异，适合前端页面兼容性与视觉回归测试，精准高效。

**协同策略**：两者搭配可覆盖功能测试与 UI 测试，提升测试效率与质量，工具易用、学习成本低，适配团队开发节奏。
